Week 3 Project Update

FIRST Resources

For the FIRST center we will be using a card reader as the main way of checking in and out of the space. Each team would need to be provided a card that would give them access to the space. As teams request cards they will be put into the Team table with information about the Team Number, Team Name, Main Contact (Name, phone number, etc.). There will be an activity log table that will track check-ins and check outs. This will be be keyed to the team so we can see which team the activity belongs to. There is the potential to be more specific than just the team and have a Team Member table. Each student could be given a card and as they request cards we store their Name, Phone Number, Address and link to the Team table so we know which team they belong to. This would require the activity table to link to the specific Team Members instead of the Teams. We can have a Kettering FIRST Student table that links Students Kettering ID and information with their FIRST team. This would allow us to give access to current kettering students through their IDs. All of this access could be limited based on a Permissions table with permission keys that allow use at different times, each user of the space would have a specific permission key linked to their information so we can limit the access on a per person level. This would be the requirements and potential requirements for the FIRST center.

Forums

Since each Kettering Student / FIRST Team Member has an ID after all the cards have been given out, they can use that ID to create an account on the Forums (will be a website). This allows us to link any Posts made to either a Kettering Student or a FIRST Team Member through their respective tables. The posts will show information about the person who made the post. There will need to be a Thread table that the posts are linked to, and they will be displayed once the student enters that respective thread. There will then need to be a Sub Forum table that the threads are linked under. There can be a Forum Permissions table for this as well to limit who can create Sub Forums, who can create Threads, and who can make Posts. This gives teams ways to communicate with one another about the space and projects going along in it. Since there is already a general FIRST forum already out on the internet. This will be the least priority of our 3 features.

Proposal System

The proposal system is a very interesting idea. This will once again require the Kettering Student ID or the FIRST Team Member ID to create a proposal (another website). There will be Proposal table where the information for each proposal is entered and tied to a User. There will be a Proposal Permission table that will allow specific users to enter proposals, and others to approve them. This could be limited to monetary proposals, or proposals for new tools for the space, but this proposal system can be expanded (relatively easily) to be a proposal system for anything throughout Kettering, club budgets, refund requests, ideas for the school and many other things. This makes it the most wide impacting and interesting of the three features.
