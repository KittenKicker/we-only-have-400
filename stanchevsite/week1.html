CONCEPT: Proposal/Idea Database

Kettering doesn’t currently (to my knowledge) have a database for collecting ideas from faculty and students about changes they would like to see on campus. From being around campus, I’ve noticed that nearly everybody I talk to has at least one thing they wish would change. One of the main challenges for administrations or even student governments is figuring out what projects to work on and what the people they represent would like them do. That is what I hope creating this database would help with.

The features for the website I’ve considered so far are fairly straight forward:

  • Login with student ID to associate each account’s activity with a specific person
  • Users should be able to browse and upvote/downvote proposals and ideas
  • Users would be allowed to submit either an informal idea (general idea that may not be fully developed)
  • Users should be able to revise their proposals/ideas into more formal plans or solutions

Given that set of features, the requirements for the project from a database standpoint would be:

  • Access to database with student login info for authentication (probably obtained through IT approval but I’m not sure about the specifics yet)
  • Table of proposals… something along the lines of:

index

title

description

user_index

num_upvote

num_downvote

index – unique key for each proposal

title – title to be displayed on the site

description – summary or additional details about the idea and/or approaches for solution

user_index – key referring to the table used for authentication to tie proposals to users

num_upvotes – int value of upvotes the idea has received

num_downvotes – int value of downvotes the idea has received

Group #4: Internet of Things(Jerry, Mikayla, Logan)


Need

Mobile devices are enabling people to become more connected to the world around them. With the movement towards an “internet of things” connectivity is being extending to household appliances, allowing consumers to monitor and control their “stuff” over the internet. As people become more connected, the demand for connected appliances will likely grow. Currently, there are only a small amount of commercially available connected appliances, many of which are still very expensive.

While this project focuses specifically on a making a connected coffee maker, it presents a framework by which a mobile web application and a back-end server could be used to control a variety of household appliances.

Approach

Several steps are needed to realize the system including modifying the coffee maker, building the back-end, and developing the mobile web application. A simple coffee maker will be modified by placing a relay in series with its ON switch. This will allow the small current from an I/O pin of a Raspberry Pi to actuate the switch inside the relay and control the much larger current required to turn the coffee maker on.

An RPi will become the “brain” of the coffee maker, using a Python program to turn the device ON/OFF and communicate with back-end to receive commands and send status information. Communication between the RPi and the back-end will be facilitated using a python implementation of the WebSocket protocol. This allows bi-directional data communication over a single TCP connection once an initial connection has been established.

The back-end will consist of a web server and a database. The web server will listen for WebSocket connections allowing commands sent from the mobile web application to be forwarded to the coffee maker. Options such as Node.js and Ruby on Rails will be considered for implementing this server. The database will store user login information and brewing history.

The mobile web app running in the mobile device’s browser will also communicate with the back-end using the WebSocket protocol. UI for the app will be implemented using HTML5/CSS3/JS. In addition, the use of a JavaScript MVC framework such as Backbone.js or Angular.js will be investigated to ease development.

Benefit

The proposed system provides several benefits. First, users can brew coffee remotely and have it ready when they return home. For example, a student walking home from class on a cold winter day could remotely initiate brewing, allowing the coffee to be ready when he/she walks in the door.

Multiple data sources such as brewing history, weather conditions, GPS location, etc. could be used to predict when a particular user might want coffee and ask them if they would like some to be brewed.

The app could also allow users to setup recurring brewing schedules, or even integrate with a user’s calendar.

Competition

After conducting online research, one commercially available internet connected coffee maker was found (http://www.mrcoffee.com/coffeemakers/BVMC-PSTX91WE.html). This product was recently released and has an MSRP of $249.99. In addition to its high cost, this product does not have all features listed above.

Jerry Brown-Lacy CS-341 Winter 2015

At Kettering university there is an abundance of housing alternatives to use as opposed to using the dorms. These options however are not very obvious and, information on these options are not readily available. My proposed web app would be used to provide students or staff to login, post information about available housing options so that students will be able to access the information at any time.

The system will enable students to post off campus housing locations. When these are posted they will also post the maximum number of residents and the number of residents living their currently. Students may then contact the landlord based on the information found on the site to see if they will be able to visit the house if any spots are still available.

Each house will have a separate section for both A and B section so that the students will be able to view the correct information at any time. The house listing will contain basic pricing information as well as a location on a map so that you know how close to campus you are located.

As a separate section of the app there will be a place for students to post about a roommate search. This feature will allow students to find potential roommates working in the same area during work term.

Users will need to be able to search for a house near campus by distance, number of open spots and cost. Users will also need to search for work term roommates by location and number of desired roommates.
